# BattleRope

ESP32 보드 기반 줄넘기 센서와 BLE로 연동하는 2인 실시간 줄넘기 대결 Android 앱입니다.

---

## 앱 소개

두 명의 플레이어가 줄넘기를 하면, ESP32 보드가 두 플레이어의 점프 횟수를 실시간으로 측정해 BLE로 앱에 전달합니다. 앱은 수신한 데이터를 콤보·피버·추월 보너스를 적용해 최종 점수로 환산하고, 두 플레이어의 대결 결과를 표시합니다.

---

## 화면 구성

### 1. Home
앱 진입점. Exercise 모드와 Competition 모드로 분기합니다.

### 2. DeviceConnection (Competition)
경쟁 모드 진입 전 BLE 기기 연결을 기다리는 화면입니다. 기기가 발견되면 자동으로 연결하고, 연결 완료 시 모드 선택으로 이동합니다.

### 3. ModeSelection (Competition)
게임 모드를 선택합니다.

| 모드 | 시간 |
|------|------|
| 스프린트 | 30초 |
| 클래식 | 60초 |
| 지구전 | 120초 |

### 4. Game (Competition)
실제 대결 화면입니다. **카운트다운(3초) → 게임 진행 → 결과** 순으로 진행됩니다. 매 초마다 점수·콤보·피버 상태·이벤트가 업데이트됩니다.

---

## 게임 구조

### 점수 계산

```
점수 증가량 = 유효 점프 수 × 콤보 배율 × 피버 배율
```

#### 콤보 티어

연속으로 점프를 성공하면 콤보가 쌓이고 배율이 올라갑니다. 줄에 걸리면 콤보가 0으로 초기화됩니다.

| 티어 | 콤보 기준 | 배율 |
|------|-----------|------|
| NONE | 0+ | x1.0 |
| BRONZE | 10+ | x1.2 |
| SILVER | 30+ | x1.5 |
| GOLD | 50+ | x2.0 |
| PLATINUM | 80+ | x2.5 |

#### 피버 타임

게임마다 랜덤한 시간대에 **20초짜리 피버 구간**이 한 번 발생합니다. 피버 중에는 모든 점수가 **2배**로 적용됩니다. 구간은 게임 시작 직후와 종료 직전을 피해 배치됩니다.

## BLE 통신 구조

Android BluetoothGatt 라이브러리를 통해 ESP32 보드와 통신합니다.

앱이 장치 이름 기반 스캔 필터로 BLE 스캔을 시작하고, 필터를 통과한 장치를 발견하면 스캔을 중단하고 GATT 연결을 맺습니다. 연결이 완료되면 해당 장치의 Service와 Characteristic을 탐색해 데이터 수신 준비를 합니다.

게임이 시작되면 운동 데이터 Characteristic의 Notify를 활성화하고, 3초 카운트다운 후 보드에 **START 명령**을 전송합니다. 보드로부터 정상 ACK를 수신하면 이후 매 초 경과 시간과 플레이어 A·B의 누적 점프 횟수가 담긴 운동 데이터가 Notification으로 전달됩니다. 앱은 이를 파싱해 콤보·피버·추월 보너스가 적용된 게임 점수로 환산합니다.

게임 도중 뒤로가기를 누르거나 시간이 종료되면 보드에 **STOP 명령**을 전송하고 운동 데이터 수신을 중단합니다.
